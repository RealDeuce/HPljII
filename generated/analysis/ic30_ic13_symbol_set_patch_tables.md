# IC30/IC13 Symbol-Set Patch Tables

Generated from the data table rooted at `0x14fce`, consumed by `0x14f16` after
built-in font map initialization.

Symbol-set names are from the LaserJet Series II Technical Reference, Table
8-1 / Table 10-2. Routine `0x14f16` only enters this path when the selected
font normalizes to symbol set `0x0115` (`8U`, HP Roman-8). It then reads the
active requested symbol-set word from `0x783144` or `0x783146` and handles it
as follows:

- `0x0005` (`0E`, HP Roman Extension): copy the upper 128 map bytes down over
  the lower 128 bytes, then clear the upper 128 bytes.
- `0x0015` (`0U`, ISO 6: ASCII): leave the lower map half alone, clear the
  upper 128 bytes.
- table hit at `0x14fce`: apply byte pairs from the selected patch table, then
  clear the upper 128 bytes.
- no table hit: leave the map as initialized by `0x14d9c`/`0x14e24`.

Each patch pair is interpreted by `0x14fa0..0x14fa4` as `map[dst] = map[src]`.
This remaps host character `dst` to the glyph index that the active map had
for character `src`.

## Example Effects on Built-In Base Maps

For the first `COURIER` and `LINE_PRINTER` records, the pre-patch base range
is `0x01..0xff`, so `map[x] = x - 1` for nonzero bytes. Under that base map, a
pair `dst,src` makes host byte `dst` select glyph index `src - 1`.
